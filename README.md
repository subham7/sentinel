# SENTINEL

> Real-time geospatial intelligence platform for monitoring active conflict theaters.

SENTINEL is a Palantir-style military intelligence dashboard that aggregates live ADS-B flight tracking, AIS vessel tracking, and OSINT incident feeds across multiple conflict theaters. It uses AI to classify events, generate hourly situation reports, and power an analyst chat interface with tool use.

---

## Features

- **Live Aircraft Tracking** — Military aircraft positions via adsb.fi + airplanes.live, updated every 30s. Callsign classification, trail rendering, strike package detection.
- **Naval Tracking** — Real-time vessel positions via AISStream WebSocket. AIS-dark detection, OFAC sanctions flagging, Strait of Hormuz status widget.
- **OSINT Incident Feed** — Events ingested from GDELT (15min), ACLED (daily), and public Telegram channels (4min). Three-pass Groq AI classifier extracts location, severity, and category.
- **Geographic Intelligence** — Military bases, nuclear sites, SAM coverage rings, shipping lanes, and chokepoint markers — all config-driven per conflict.
- **AI Situation Reports** — Hourly sitreps generated by `llama-3.3-70b-versatile` via Groq, summarising aircraft posture, vessel activity, and recent incidents.
- **Analyst Chat** — Claude `claude-sonnet-4-6` with tool use. Ask questions in plain English; the model queries live incidents, aircraft, vessels, and theater posture scoped to the active conflict.
- **Signal Convergence** — Browser-side 0.5°×0.5° grid detects when aircraft, vessels, and incidents cluster in the same cell simultaneously.
- **Surge Detection** — Welford online algorithm maintains a 30-day baseline in localStorage and surfaces Z-score anomalies in aircraft and vessel counts.
- **Economic Layer** — EIA Brent/WTI oil prices with 60-day sparkline, Iranian Rial parallel market rate scraped from bonbast.com.
- **Config-driven conflicts** — Adding a new theater requires one `ConflictConfig` object. Workers, API routes, and the frontend all derive from it automatically.

## Getting Started

### Prerequisites

- Node.js 22+
- npm 10+

### Install

```bash
git clone https://github.com/your-username/sentinel.git
cd sentinel
npm install
```

### Environment Variables

Copy the template and fill in your keys:

```bash
cp .env.example .env
```

```bash
# ── AI ─────────────────────────────────────────────────────────────────────
GROQ_API_KEY=              # groq.com — free tier, 14,400 req/day
ANTHROPIC_API_KEY=         # anthropic.com — analyst chat

# ── Cache ──────────────────────────────────────────────────────────────────
UPSTASH_REDIS_REST_URL=    # upstash.com — free tier available
UPSTASH_REDIS_REST_TOKEN=

# ── Flight Tracking ────────────────────────────────────────────────────────
# adsb.fi requires no auth (primary source)
OPENSKY_USERNAME=          # optional
OPENSKY_PASSWORD=          # optional

# ── Vessel Tracking ────────────────────────────────────────────────────────
AISSTREAM_API_KEY=         # aisstream.io — free, register at site

# ── Incident Data ──────────────────────────────────────────────────────────
ACLED_EMAIL=               # acleddata.com — free academic/research access
ACLED_PASSWORD=

# ── Economic ───────────────────────────────────────────────────────────────
EIA_API_KEY=               # eia.gov/opendata — free

# ── Telegram ───────────────────────────────────────────────────────────────
TELEGRAM_CHANNELS_US_IRAN=         # comma-separated public channel usernames
TELEGRAM_CHANNELS_ISRAEL_GAZA=

# ── Server ─────────────────────────────────────────────────────────────────
PORT=3001
ALLOWED_ORIGINS=http://localhost:3000
NODE_ENV=development
```

**Minimum keys to see data:**
- Without any keys: static overlays, map layers, and UI are fully functional
- `UPSTASH_REDIS_REST_URL` + `TOKEN`: enables caching (falls back to in-memory without it)
- `GROQ_API_KEY`: enables AI classification and sitreps
- `AISSTREAM_API_KEY`: enables live vessel tracking
- `ANTHROPIC_API_KEY`: enables analyst chat

### Run

```bash
# Start both frontend and API in parallel
npm run dev

# Or individually
npm run dev --workspace=apps/sentinel-fe   # → http://localhost:3000
npm run dev --workspace=apps/sentinel-api  # → http://localhost:3001
```

## Deployment

### Frontend → Vercel

1. Import repo on [vercel.com](https://vercel.com)
2. Set **Root Directory** to `apps/sentinel-fe`
3. Add env var: `NEXT_PUBLIC_API_URL=https://your-api.railway.app`
4. Deploy

### API → Railway

1. New project on [railway.app](https://railway.app) → deploy from GitHub
2. Go to **Settings → Build** → set Dockerfile path to `apps/sentinel-api/Dockerfile`
3. Add a **Volume** mounted at `/data` (persists SQLite)
4. Add all environment variables from the table above
5. Set `ALLOWED_ORIGINS=https://your-app.vercel.app`

## API Endpoints

```
GET  /health
GET  /api/conflicts                          → ConflictConfig[] + live stats
GET  /api/conflicts/:slug                    → Single conflict config
GET  /api/conflicts/:slug/aircraft           → Aircraft[]
WS   /ws/conflicts/:slug/aircraft            → Aircraft[] push every 5s
GET  /api/conflicts/:slug/vessels            → Vessel[]
WS   /ws/conflicts/:slug/vessels             → Vessel[] push every 10s
GET  /api/conflicts/:slug/incidents          → Incident[] (params: hours, severity)
SSE  /api/conflicts/:slug/incidents/stream   → Live incident stream
GET  /api/conflicts/:slug/sitrep             → AI situation report (cached 1h)
POST /api/conflicts/:slug/analyst-chat       → Claude tool-use chat
GET  /api/economic/oil                       → Brent + WTI prices
GET  /api/economic/rial                      → USD/IRR parallel market rate
```

## Architecture Notes

- **Config-driven**: all conflicts, data sources, overlays, and map bounds are defined in `packages/shared/src/conflicts.ts`. Zero hardcoding in workers or components.
- **Stale > blank**: every endpoint returns cached data even when upstream APIs are down. Priority: fresh cache → stale cache (24h) → empty array. Never a 500 to the frontend.
- **Client-side intelligence**: convergence detection, surge analysis, and strike package detection all run in the browser. The backend only proxies and stores raw data.
- **Three-tier cache**: Redis (fresh) → Redis stale key (24h) → in-memory fallback. Works without Redis credentials in development.
- **Circuit breakers**: each upstream API has its own circuit breaker (3 failures → 5 min cooldown).
